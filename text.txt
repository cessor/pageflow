1. Einführung
=============
Zur Einführung sehen Sie einige Erklärungen eingeblendet. Lesen Sie sich alles in Ruhe durch. Während der eigentlichen Erhebung werden die Hinweise nicht eingeblendet.

[Weiter]

2. Erhebung
===========
Hier sehen Sie eine in C# geschriebene Funktion. *Hinweis: Nicht aufregen*. Der Code ist kommentiert. Der Zweck der Funktion steht in der ersten Kommentarzeile. Die restlichen Zeilen beinhalten eine Übersicht der im Code verwendeten Variablen.

[Weiter]

3. Ihre Aufgabe
===============
Ihre Aufgabe ist es, einen Fehler im Code zu finden.
Der Code kompiliert korrekt, aber er weist zur Laufzeit ein fehlerhaftes Verhalten auf. Jede Funktion hat nur einen Fehler.

[Weiter]

4. Fenster
==========
Während der Studie sind vom Code sind nur einige Zeilen sichtbar, die restlichen sind ausgeblendet. Sie können den sichtbaren Bereich mit den Pfeiltasten auf Ihrer Tastatur verschieben.

    {KeyBoard}

<Verschieben>

5. Fehler gefunden
==================
Sobald Sie den Fehler gefunden haben, dann drücken Sie bitte die Leertaste.
Zur Erinnerung wird die Steuerung am unteren Bildschirmrand eingeblendet.

    {KeyBoard}

<Leertaste>

6. Zeilennummer
===============
Wenn Sie den Fehler gefunden haben, dann geben Sie bitte die Zeilennummer an, in der sich der Fehler befindet.

[Weiter]

7. Beschreibung
===============
Geben Sie bitte auch eine kurze Beschreibung des Fehlers an und erklären Sie, was genau falsch ist.

[Weiter]

8. Abschluss
============
Zum Abschluss des Durchgangs klicken Sie auf den "Gefunden"-Button. Durch den "Abbrechen" Button kehren Sie zum Code zurück.

[Weiter]

9. Aufgeben
===========
Wenn Sie den Fehler nicht finden können, dann drücken Sie auf "Aufgeben".

[Weiter]

10. Zeitbegrenzung
==================
Sie haben für das Auffinden des Fehlers 10 Minuten Zeit. Nach Ablauf der Zeit wird automatisch das nächste Problem angezeigt.

[Weiter]

11. Ablauf
==========
Insgesamt sehen Sie $number verschiedene Codes. Jeder behandelt ein anderes Problem. Zwischen den Durchgängen können Sie eine kurze Pause einlegen. Ein Durchgang dauert ca. 5 - 10 Minuten.

[Weiter]

12. Zusammenfassung
===================
Zusammenfassend ist Ihre Aufgabe also:
 \- Aufgabe: Finden Sie einen semantischen Fehler
 \- Verschieben Sie den Code mit den Pfeiltasten
 \- Drücken Sie die Leertaste, wenn Sie den Fehler gefunden haben.

[Weiter]

13. Start
=========
Das Tutorial ist nun beendet. Es folgen nun $number echte Durchgänge.

Achten Sie bitte auf eine störungsfreie Umgebung. Störungen können die Daten unbrauchbar machen. Schalten Sie ihr Handy aus, schließen Sie alle Messenger und Outlook und bitten Sie Ihre Kollegen, Sie nicht zu stören.

[Start]

14. Es folgt Durchgang $current von $number. (:first)
-----------------------------------------------------
Drücken Sie die Leertaste um zu beginnen.

<Leertaste>

15. Es folgt Durchgang $current von $number. (:next)
----------------------------------------------------
Wenn Sie möchten, können Sie eine kurze Pause einlegen.
Drücken Sie die Leertaste um zu beginnen.

<Leertaste>

16. Fast fertig!
================
Um die Daten besser bewerten zu können, werden noch einige Angaben benötigt.

[Weiter]
